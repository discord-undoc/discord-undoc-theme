Reference

Currently the preprocessor only works with 5 types of custom elements
They are:

  • Enpoints
  • Indicators
  • Alert Boxes
  • User Badges
  • Spoilers

Endpoints

There are upto 9 elements which are considered as “endpoint” elements
They are the 9 methods: get head post put delete connect options trace patch

<b class="endpoint"><span class="endpoint-c"><span class="http-method get">GET</span> /some/endpoint</span></b>\
<b class="endpoint"><span class="endpoint-c"><span class="http-method head">HEAD</span> /some/endpoint</span></b>\
<b class="endpoint"><span class="endpoint-c"><span class="http-method post">POST</span> /some/endpoint</span></b>\
<b class="endpoint"><span class="endpoint-c"><span class="http-method put">PUT</span> /some/endpoint</span></b>\
<b class="endpoint"><span class="endpoint-c"><span class="http-method delete">DELETE</span> /some/endpoint</span></b>\
<b class="endpoint"><span class="endpoint-c"><span class="http-method connect">CONNECT</span> /some/endpoint</span></b>\
<b class="endpoint"><span class="endpoint-c"><span class="http-method options">OPTIONS</span> /some/endpoint</span></b>\
<b class="endpoint"><span class="endpoint-c"><span class="http-method trace">TRACE</span> /some/endpoint</span></b>\
<b class="endpoint"><span class="endpoint-c"><span class="http-method patch">PATCH</span> /some/endpoint</span></b>

Alert Boxes

These are those highlighted boxes which are some pieces of text that the user must read.
There are 3 types of alert boxes: note info warn

<div class="note"><p>This is a random note.</p></div>
<div class="info"><p>You know what this reminds me of? Blobs!</p></div>
<div class="warn"><p>┻━┻ ︵ヽ(`Д´)ﾉ︵﻿ ┻━┻</p></div>

User Badges

These are cute looking badges that link to users on discord
You can create one like this:

<a class="user" href="https://discord.com/users/1234"><img src="https://cdn.discordapp.com/embed/avatars/3.png" alt="Deleted User's avatar" class="avatar" width="25px" height="25px">Deleted User<span>#0000</span></img></a>

The id attribute should be the ID of that user.

Spoilers

Inspired from discord’s spoilers, discord-undoc also has spoilers
They can be defined like so:

<input type="checkbox" class="spoiler" id="spoiler-6921326521977"><label class="spoiler" for="spoiler-6921326521977">spoiler'd content</label>
