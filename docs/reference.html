Reference

Structure

Any custom element can be divided into 2 parts

  1. The type
  2. The content

Some elements do not require any content.

The format is: {::(TYPE) (CONTENT)}

  • The TYPE and the CONTENT must be separated by a space ( )
  • Passing in content for an element that doesnâ€™t accept any may result in rendering issues!
  • Element types are case sensitive.

Formats and Examples

Custom elements are manually added, hence only the elements listed here work, others will end-up w/ faluty rendering

User Badge:

  • Format: {::user (USERNAME#DISCRIMINATOR)-(USER_ID)-(AVATAR_HASH)}
  • Example: {::user Rydix#1726-755792681313108018-a_90be0df51f8cf7dc36bec9cb1ad1459c} Rydix#1726

HTTP Stuff:

  • Format: {::(get|head|post|put|delete|connect|options|trace|patch) (ENDPOINT)}
  • Exaple: {::post /auth} POST /auth

Indicators:

  • Format: {::(undoc|nobot|iandeploy)}
  • Example: {::iandeploy} rocket_launch

Alert Boxes:

  • Format: {::(note|info|warn) (CONTENT)}
  • Exaple: {::note Hi!}

    Hi!


Accordions:

  • Format: {::(details|details-open) (SUMMARY-optional)<summ>-optional(CONTENT)}
  • Example: {::details-open Hello}

    Hello

Spoilers:

  • Format: {::spoiler (CONTENT)}
  • Example: {::spoiler this is a spoiler}
